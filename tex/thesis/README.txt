--- 論文を書くとき------------------------------------------------
 TEX ファイルを編集していけば大丈夫です
 デフォルトの状態では

  thesis.tex : 論文の全体
  intro.tex  : 序論
  base.tex   : 基礎理論の全体
  method.tex : 提案手法の全体
  exp.tex    : 評価実験の全体
  concl.tex  : 結論
  ack.tex    : 謝辞
  ref.tex    : 参考文献

になっています

また base, method, exp の各ディレクトリにそれぞれの節が入ってます

 Latex の文法とかは自分で調べてください
 ファイルやディレクトリの名前は好きに変えたり加えたりしてください
※  TEX ファイルは たぶん EUC じゃないと文字化けします

図などの画像は figure ディレクトリに入れてください
また Latex では eps の画像しか使えないので変換してください
変換方法は下に書いてます
-------------------------------------------------------------------

--- eps への変換 ----------------------
 $convert [input-file] [output-file]
 e.g.) $convert piano.jpg piano.eps

複数ファイルの場合
 $mogrify [options] [input-file]
 e.g.) $mogrify -format eps piano.jpg

※make にも用意してます
---------------------------------------

--- 文字コードの確認 --------------------------------------------------------------------------------
 $platex --version

と入力することで最初の行に(EUC)とか(SJIS)と表示されるので TEX ファイルをその文字コードに変換すること
※makeにも用意してます

※ そんなの表示されないという人はそもそもplatexが入ってないので入れてください
※ Ubuntu ソフトウェアセンターにて latex で検索をかけて
※ "Texmaker(Latex開発環境)"をインストールすれば大丈夫だと思います
-----------------------------------------------------------------------------------------------------

--- 文字コードの変換方法 -----------------------------------

※一応コンパイルする時に文字コードの変換は行うようにしてます

<--- emacs を使うパターン ------------------->

 emacs上で

C-x [RET] f
か
ESC-x set-buffer-file-coding-system

と入力するとemacsの下部に

Coding system for saving file (default nil):

と表示されるので

EUC  なら euc-japan
JIS  なら junet
SJIS なら sjis

と入力する
<-------------------------------------------->

<--- nkf コマンドを使うパターン--->

EUC なら
$nkf -e --overwrite [file]

JIS なら
$nkf -j --overwrite [file]

SJISなら
$nkf -s --overwrite [file]

 と入力する
